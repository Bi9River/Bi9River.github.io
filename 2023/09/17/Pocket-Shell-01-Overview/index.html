# Pocket Shell 01 Overview

This is the overview of the Pocket Shell system. I drew a N-Tier layer diagram to show the overall structure of the system.

From the diagram, the system can be divided into some modules.

  • Game Client: The mobile app.
  • Web User Account Management Frontend: A website for users to:
    • Register.
    • Edit their profile, such as avatar, nickname...
    • Manage the friend list, such as adding/removing friend.
  • Game Server: Synchronizes the in-game user position.
  • Web User Account Management Backend
  • Chat System Server: In-game text chatting server. It is separated from the game server.
  • DBs & Connectors

The system is designed using a microservices architecture approach. Within this framework, distinct components such as the Game Client, Web User Management Frontend, Game Server, Web User Management Backend, and Chat System Server are isolated from one another. Each of these components maintains its own independent data, avoiding shared storage.

This modular approach allows for a refined decomposition of services within what would otherwise be a more complex system, ensuring the decoupling of services based on their responsibilities.
